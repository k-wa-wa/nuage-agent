# エージェント自己改善フローの仕組みと使い方

\`nuage-agent\` は、各自律エージェントの挙動を継続的かつ自律的に洗練させるための **自己改善フロー (Self-Improvement Flow)** を備えている。

この仕組みは、過去にエージェントが実行した際の詳細な標準出力・エラー出力ログを Claude Code CLI (\`claude\`) に読み込ませ、ログ内の失敗パターンや非効率な動きを分析して、\`src/agents/\` 配下にあるプロンプト定義（\`buildPrompt\` 等）やロジック自体を自動で改善（自己修正）する機能である。

---

## 1. 処理の流れ

自己改善コマンドを実行すると、システムは以下のプロセスを自動的に実行する。

\`\`\`
[ログの蓄積] workspaces/_/logs/_.log
│
▼ (1. 走査 & 一覧抽出)
self-improve スクリプト (src/runner/bin/self-improve.ts)
│
▼ (2. 指示ファイル作成)
self_improvement_prompt.md (ルートディレクトリ)
│
▼ (3. エージェント起動)
Claude Code CLI (claude --dangerously-skip-permissions)
│
├─► [ログの動的読込] 各 .log ファイルの分析
├─► [コード修正] src/agents/ 配下のファイルを改善
└─► [後片付け] 改善済みのログと指示ファイルを削除
\`\`\`

1. **ログファイルの走査**: \`workspaces/\*/logs/\` ディレクトリ配下にあるすべての \`.log\` ファイルを走査し、ファイルパスのリストを取得する。
2. **指示ファイルの生成**: ルートディレクトリに \`self_improvement_prompt.md\` という一時ファイルを生成する。このファイルには、自己改善を実行するための指示文と、検出されたログファイルへの相対・絶対パスのリストが記載される。
3. **Claude CLI の起動**: \`claude\` を \`--dangerously-skip-permissions\` フラグ付きで起動し、一時ファイルを読み込ませて自己改善タスクを指示する。
4. **自己改善の実行**:
   - 起動したエージェントは、指示ファイルにリストされたログファイルの中身を動的に読み込み、エラーや非効率なコマンド実行、プロンプトの記述漏れがないか分析する。
   - 分析結果に基づいて、\`src/agents/\` 配下のエージェントのソースコード（\`agent.ts\`）を直接編集し、対策やプロンプト指示の強化を行う。
5. **クリーンアップ**:
   - 改善が正常に完了したログファイルをディスクから削除する。
   - 最後に、一時指示ファイル \`self_improvement_prompt.md\` を削除して終了する。

---

## 2. 重要な設計ルール

### A. Git操作の禁止

自己改善フローを実行するエージェント（Claude Code CLI）には、**「Git操作（git commit, git push, ブランチ作成・切替など）を行ってはならない」**という強い制約をプロンプトで指示している。
これは以下の理由によるものである：

- 意図しないコミットが自動生成され、Git履歴が汚れるのを防ぐため。
- 開発者が実際にコードを確認（\`git diff\`）し、動作検証を行ってからコミット・マージを行えるようにするため。
- したがって、エージェントは作業ディレクトリの変更（Working Tree）のみを行い、最終的なGitのコミットやPR作成は人間（開発者）が責任を持って行う。

### B. 改善対象の制限 (被開発リポジトリは対象外)

エージェントは、**ログに記録されている被開発・被テスト対象リポジトリ（例: \`workflow-sandbox\`）のソースコードやバグを直接修正してはならない**。
改善の対象は、あくまで**この開発パイプラインツール自体（\`nuage-agent\` および \`src/agents/\` 配下のエージェントのプロンプト・挙動定義）のみ**である。対象リポジトリ側のバグ修正は本タスクの目的外である。

## 3. コマンドラインでの使い方

自己改善フローを実行または確認するための以下の npm スクリプトが提供されている。

### A. ドライラン（プロンプト生成のみ）

実際に Claude CLI を起動せず、どのログが対象になりどのような指示が送られるかを確認したい場合に使用する。

\`\`\`bash
bun run self-improve:dry
\`\`\`

実行後、プロジェクトルートに \`self_improvement_prompt.md\` が生成される。内容に問題がないかエディタ等で確認すること。

### B. 自己改善フローの実行

実際に自己改善フローを開始する。エージェントが起動し、ログファイルの分析とコードの修正を自律的に進める。

\`\`\`bash
bun run self-improve
\`\`\`

> [!NOTE]
> 起動した Claude Code は \`--dangerously-skip-permissions\` フラグにより、自動的にファイルの読込・書込・削除を実行する。ログ量が多い場合は実行に数分かかることがある。終了後、\`git diff\` でエージェントがどのように改善したかを確認すること。

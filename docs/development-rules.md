# 開発ルール (Development Rules)

\`nuage-agent\` プロジェクトにおける開発ルールおよびスタイル規約を以下に定める。本リポジトリに貢献する開発者およびエージェントは、本ルールを遵守すること。

---

## 1. 記述スタイル（常体の統一）

- **プロンプトおよびドキュメントは、すべて常体（「だ・である」調）で記述すること。**
- 敬体（「です・ます」調）は使用しないこと。

### 適用範囲

- **LLMエージェントへの指示プロンプト**（\`src/agents/\*_/_.ts\` の \`buildPrompt\` メソッド内の指示文など）
- **ドキュメント類**（\`docs/\*_/_.md\`、\`README.md\`、各種仕様書など）

### 例

- **不可（敬体）**:
  - `〜を分析してください。`
  - `〜を実行します。`
- **可（常体）**:
  - `〜を分析せよ。` / `〜を分析すること。`
  - `〜を実行する。`

---

## 2. ローカル検証ルール

- **コミットやプッシュを行う前に、必ず以下のコマンドを実行してローカルで正常にビルド・検証できることを確認すること**
  - ビルドおよび静的解析チェック: `bun build:ci`
  - 単体テストの実行: `bun test`
